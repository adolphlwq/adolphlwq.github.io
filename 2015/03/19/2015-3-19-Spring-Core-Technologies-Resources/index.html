6.3 Built-in Resource implementations（P138）

ClassPathResource

这里的类代表了一种能从classpath里获得的资源，它使用线程上下文的类加载器、给定的类加载器、给定的类来加载资源

如果classpath路径的资源存在于本机的文件系统的话，Resource实现的方法支持Java.io.File里面的处理方法(个人理解)，但是Resourcebu不支持classpath里jar包里的文件或者是那些没有被扩充的到文件系统的文件。为了解决这个问题，Resource总是支持java.net.URL.

ClassPathResource被java里的ClassPathResource明确建立，当你调用包含一个代表路径的字符串参数的API方法时，ClassPathResource会被隐含建立。对于后者，JavaBeans PropertyEditor会识别参数路径里的classpath:前缀，并创建ClassPathResource。

FileSystemResource

它是java.io.File的实现，支持File和URL

ServletContextResource

是ServletContext资源的方法实现，interpreting relative paths within the relevant web application's root directory.指相关web应用根目录下的相对路径。支持流访问和URL访问。

InputStreamResource

ByteArrayResource

6.4 The ResourceLoader（P140）

6.6 Resources as dependencies

资源依赖注入并不复杂，只要Beans中有Resource objects,Spring容器会自动转换路径并配置进去。注意这里的路径没有前缀，因为web应用中应用上下文自身会被用作ResourceLoader，xml文件会被ClassPathResource，FileSystemResource,ServletContextResource根据情况加载。可以添加前缀强迫使用相关的类加载器
