git-remote-branch

发表于 2015-03-26   |  


branch

preiew knowlege

git是一款优秀的版本控制工具，分支是其强大的特性之一。

git用reposity（仓库）来控制版本。git可以搭建在本地，也可以搭建在远程的服务器上。github.com是最大的公开的远程仓库（当然github上也可以建立收费的私人仓库）。git系统是开源的，你也可以在自己的内网服务器上搭建自己的github远程git系统。

git通过分支来进行不同需求的开发。一般有如下几个分支：

  • master……主分支，上面的代码是稳定版本
  • dev………也叫develop版本，开发版，一般开发好测试好之后再合并到主分支上发布
  • feature…..特性分支，有什么新的特性或需求可以在这个分支开发
  • issue…….别人针对项目提出的issue(问题或者bug)，新建issue分支来修补bug
  • …还有很多自定义的分支

remote branch

如果是个小项目，一个人就能搞定，不需要团队合作，那么只需要在自己的电脑管理版本。

如果是个大的开源项目,Spring、Linux内核、Struts等需要多人合作且开源的项目，就需要放到远程服务器上。这样就有了远程分支和本地分支的概念！

假设的情景：我的计算机的仓库

github官网和自己计算机对应的远程仓库

别人的仓库分支：

Look Out！本地仓库是本地仓库，远程仓库是远程仓库，如果两者不关联，那他们之间没什么关系。

git clone https://github.com/adolphlwq/sushems.git 默认在当前文件夹克隆项目的仓库,生成sushems文件夹，项目的所有信息在sushems中。使用git clone,本地仓库和远程仓库是对应的，且本地分支默认为master,远程分支默认为origin.并且本地分支master默认跟踪远程master分支。

branch命令（直接从manual复制过来的我会乱说！manual is your best frend!）

用法：

git branch [选项] [-r | -a] [--merged | --no-merged]
   或：git branch [选项] [-l] [-f] <分支名> [<起始点>]
   或：git branch [选项] [-r] (-d | -D) <分支名>...
   或：git branch [选项] (-m | -M) [<旧分支>] <新分支>

通用选项

-v, --verbose         显示哈希值和主题，若参数出现两次则显示上游分支
-q, --quiet           不显示信息
-t, --track           设置跟踪模式（参见 git-pull(1)）
--set-upstream        改变上游信息
-u, --set-upstream-to <upstream>
                      change the upstream info
--unset-upstream      Unset the upstream info
--color[=<何时>]      使用彩色输出
-r, --remotes         作用于远程跟踪分支
--contains <提交>     只打印包含该提交的分支
--abbrev[=<n>]        用 <n> 位数字显示 SHA-1 哈希值

具体的 git-branch 动作：

-a, --all             列出远程跟踪及本地分支
-d, --delete          删除完全合并的分支
-D                    删除分支（即使没有合并）
-m, --move            移动/重命名一个分支，以及它的引用日志
-M                    移动/重命名一个分支，即使目标已存在
--list                列出分支名
-l, --create-reflog   创建分支的引用日志
--edit-description    标记分支的描述
-f, --force           强制创建（当已经存在）
--no-merged <提交>    只打印没有合并的分支
--merged <提交>       只打印合并的分支
--column[=<风格>]     以列的方式显示分支

上面的都比较常用。

我之前纠结的地方在于本地分支和远程分支在修改、提交的关系，现在说说我的理解

  • 远程分支是不能在本地实时编辑同步的
  • 只能开发提交代码到本地分支
  • 本地分支可以push到远程分支，这是他们建立联系的地方
  • git branch remote-name/branch-name来新建远程分支
    • git branch origin/dev
  • git branch branch-name新建本地分支
    • git branch dev
  • 这时dev和origin/dev是没有联系的，git push origin dev会报错，因为我们没有指定本地dev分支要提交的远程分支是谁！
  • git branch —set-upstream-to=remote-name/branch-name local-beanch-name
    • git branch —set-upstream-to=origin/dev dev
    • 建立联系后还要git pull来更新
  • 或者用另一种方法：git branch —set-upstream local-branch-name remote-name/remote-branch-name
    • git branch —set-upstream dev origin/dev

更新命令

  • git merge origin/serverfix…[将远程分支合并到当前分支]
  • git checkout -b dev origin/dev…[这会切换到新建的 serverfix 本地分支，其内容同远程分支 origin/serverfix 一致]
  • git checkout —track origin/dev…[同上]

从远程分支 checkout 出来的本地分支，称为 跟踪分支 (tracking branch)。跟踪分支是一种和某个远程分支有直接联系的本地分支。在跟踪分支里输入 git push，Git 会自行推断应该向哪个服务器的哪个分支推送数据。同样，在这些分支里运行 git pull 会获取所有远程索引，并把它们的数据都合并到本地分支中来。

  • git push origin :[remote branch name]…[删除远程分支名]
