pre-knowlege

HTML

  • HTML 是用来描述网页的一种语言
  • HTML 指的是超文本标记语言 (Hyper Text Markup Language)
  • HTML 不是一种编程语言，而是一种标记语言 (markup language)
  • 标记语言是一套标记标签 (markup tag)
  • HTML 使用标记标签来描述网页

CSS

  • Cascading[kæ’skeɪdɪŋ] Style Sheets：层叠样式表、级联样式表
  • 样式定义如何显示 HTML 元素、优化网页的表现形式
  • 样式通常存储在样式表中
  • 把样式添加到 HTML 4.0 中，是为了解决内容与表现分离的问题
  • 外部样式表通常存储在 CSS 文件中

XML

  • XML 指可扩展标记语言（EXtensible Markup Language）
  • XML 是一种标记语言，很类似 HTML
  • XML 的设计宗旨是传输数据，而非显示数据
  • XML 标签没有被预定义。您需要自行定义标签。
  • XML 被设计为具有自我描述性。
  • XML 是 W3C 的推荐标准

简介

主流笔记本的硬盘大小在500G、750G和1T,为了合理、有效的使用磁盘，对其进行科学的分区（即分成你们熟悉的C、D、E盘等），下次安装软件时，修改安装路径到软件盘就好了

盘符 大小 用途
C 100-150G 系统盘，存放操作系统相关文件
D 100G左右 存放所有需要安装的软件

这里的分区不是绝对的，大家可以根据自己的硬盘大小自行调整，但是系统盘至少要100G

WEB-INF文件夹

  1. WEB-INF文件夹是在tomcat中部署web应用必须的文件夹，没有它web应用无法运行。
  2. 它是一个安全目录，即只有服务端才可以访问，客户端无法访问。
  3. web.xml:非常重要的文件，里面可以配置应用的默认首页，不同业务逻辑调用的不同接口（浏览器浏览器点击登录，后台决定哪个代码负责处理）等。
  1. lib:存放java代码需要的jar包
  2. classes:存放编译好的.class文件

myeclipse下更改项目虚拟目录

  1. 右击项目名
  2. preperties
  3. myecipse-web

Tomcat初步

发表于 2015-01-15   |   分类于 Java   |  

安装

tomcat是apache软件基金组织的一款开源项目，去apache官网http://apache.org/找就可以找到，主要有安装和解压版，推荐使用解压版，解压好之后自己配置端口、用户名和密码等信息。

tomcat的文件目录

  1. /bin Startup, shutdown, and other scripts. The .sh files (for Unix systems) are functional duplicates of the .bat files (for Windows systems). Since the Win32 command-line lacks certain functionality, there are some additional files in here.(启动、关闭和其他脚本。.sh脚本适用于unix系统，.bat脚本适用于windows系统，他们的功能相同。由于com32的命令行缺少某功能，bin目录增加了额外的文件)
  2. /conf - Configuration files and related DTDs. The most important file in here is server.xml. It is the main configuration file for the container.(配置文件和相关的dtd文件，最重要的是server.xml文件，它是容器的主要配置文件)
  3. /logs - Log files are here by default.（日志文件，tomcat运行报错或者部署的项目报错时，这个文件夹是必须看的。）
  4. /webapps - This is where your webapps go.(你web应用部署的文件夹)
  5. /lib tomcat用到的库
  6. /work tomcat解析部署项目时的缓存文件
